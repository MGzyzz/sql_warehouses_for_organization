1)Таблица warehouses(Склады):

Таблица warehouse нужна для хранение списков складов

id serial primary key - создает автоматический индивидуальный id ключ

name varchar(255) not null - создает поля name для хранение имени склада. varchar - строковое значение. Общая количество допустимых букв 255. not null - говорит о том что таблица должна быть заполнена обязательно заполнена или же значение не должно быть null

2)Таблица categories(Категории):

Таблица categories нужна для хранение списка категории 

id serial primary key -  создает автоматический индивидуальный id ключ

name varchar(255) not null - создает поля name для хранение имени категории. varchar - строковое значение. Общая количество допустимых букв 255. not null - говорит о том что таблица должна быть заполнена обязательно заполнена или же значение не должно быть null

3)Таблица brands(брэнды):

Таблица brands нужна для хранение списка брэндов

id serial primary key - создает автоматический индивидуальный id ключ

name varchar(255) not null -  создает поля name для хранение имени брэндов. varchar - строковое значение. Общая количество допустимых букв 255. not null - говорит о том что таблица должна быть заполнена обязательно заполнена или же значение не должно быть null

4)Таблица products(продукты, товары):

Таблица products нужна для хранение списка продуктов, товаров, категории и брэндов

id serial primary key - создает автоматический индивидуальный id ключ

name varchar(255) not null - создает поля name для хранение имени продуктов, товаров. varchar - строковое значение. Общая количество допустимых букв 255. not null - говорит о том что таблица должна быть заполнена обязательно заполнена или же значение не должно быть null

category_id int not null references categories(id) - устанавливаем связь с таблицой categories с её id. int - числовое значаение. not null - говорит о том что таблица должна быть заполнена обязательно заполнена или же значение не должно быть null

brand_id int not null references brands(id) - устанавливаем связь с таблицой brands с её id. int - числовое значаение. not null - говорит о том что таблица должна быть заполнена обязательно заполнена или же значение не должно быть null

5)Таблица contractors(контрагенты):
Таблица contractors нужна для хранение списка контрагентов, товаров

id serial primary key - создает автоматический индивидуальный id ключ

name varchar(255) not null - создает поля name для хранение имени контрагентов. varchar - строковое значение. Общая количество допустимых букв 255. not null - говорит о том что таблица должна быть заполнена обязательно заполнена или же значение не должно быть null

6)warehouse_products(склады с продуктами):

Таблица warehouse_products - нужна для хранение списке о том где находится указанный продукт на каком складе и какое количество их

id serial primary key - создает автоматический индивидуальный id ключ

warehouse_id int not null references warehouses(id) - устанавливаем связь с таблицой warehouse с её id. int - числовое значаение. not null - говорит о том что таблица должна быть заполнена обязательно заполнена или же значение не должно быть null

products_id int not null references products(id) - устанавливаем связь с таблицой products с её id. int - числовое значаение. not null - говорит о том что таблица должна быть заполнена обязательно заполнена или же значение не должно быть null

quantity int not null check (quantity >= 0) - данный столбец нужен для хранение данных о количества продуктов. int - числовое значение. not null - говорит о том что таблица должна быть заполнена обязательно заполнена или же значение не должно быть null. check - своего рода проверка которая говорит о том что quantity не должен быть меньше 0, но может быть равен 0

7)transition(транзакции, переход, перемещение) - предназначена для хранение информации о транзакциях товаров между контрагентами и складами. Так же в ней показано время когда произошли транзакции:

id serial primary key  - создает автоматический индивидуальный id ключ

timestamp timestamp not null default  now() - создает столбец для хранение времени, default now - устанавливает значение по умолчанию настоящие время

product_id int not null references products(id) - устанавливаем связь с таблицой products с её id. int - числовое значаение. not null - говорит о том что таблица должна быть заполнена обязательно заполнена или же значение не должно быть null

contractor_id int not null references contractors(id) - устанавливаем связь с таблицой contractors с её id. int - числовое значаение. not null - говорит о том что таблица должна быть заполнена обязательно заполнена или же значение не должно быть null

warehouse_from_id int not null references warehouses(id) - устанавливаем связь с таблицой warehouses с её id. int - числовое значаение. not null - говорит о том что таблица должна быть заполнена обязательно заполнена или же значение не должно быть null

warehouse_to_id int not null references warehouses(id) - устанавливаем связь с таблицой warehouses с её id. int - числовое значаение. not null - говорит о том что таблица должна быть заполнена обязательно заполнена или же значение не должно быть null

quantity int not null check ( quantity > 0) - данный столбец нужен для хранение данных о количества продуктов. int - числовое значение. not null - говорит о том что таблица должна быть заполнена обязательно заполнена или же значение не должно быть null. check - своего рода проверка которая говорит о том что quantity не должен быть меньше 0
